# Mixed Traffic Test (혼합 트래픽 테스트)

## 1. 테스트 목적

본 테스트는 빠른 요청과 느린 요청이 혼재된  
실제 서비스 환경과 유사한 트래픽 패턴에서  
시스템 성능 특성을 검증하기 위해 수행되었다.

특히, 소수의 느린 요청이 전체 시스템 성능과  
사용자 경험에 미치는 영향을 확인하는 것을 목표로 한다.

---

## 2. 테스트 대상 및 환경

- 테스트 대상(Host): https://httpbin.org
- 테스트 유형: Mixed Traffic Test
- 동시 사용자 수: 200명
- Ramp-up: 초당 30명
- 트래픽 구성 비율:
  - 빠른 요청: 70%
  - 일반 요청: 20%
  - 느린 요청: 10%

---

## 3. 혼합 트래픽 설계 의도

실제 서비스에서는 모든 요청이 동일한 비용을 가지지 않는다.

- 대부분의 요청은 빠르게 처리됨
- 일부 요청은 외부 연동, 복잡한 처리로 인해 지연 발생
- 느린 요청이 시스템 큐를 점유하여  
  전체 응답 시간에 영향을 미칠 수 있음

본 테스트는 이러한 현실적인 상황을 재현하기 위해 설계되었다.

## 4. 주요 관찰 지표

- 50% percentile(Median) 응답 시간
- 95% percentile 응답 시간
- Failures 비율
- RPS와 사용자 수 대비 처리 효율

---

## 5. 결과 해석

테스트 결과 다음과 같은 현상이 관측되었다.

- Median 응답 시간조차 SLA를 크게 초과
- 95% percentile 응답 시간이 수십 초 단위로 증가
- Failures 비율이 60% 이상으로 상승
- RPS가 사용자 수 대비 매우 낮게 유지

이는 서버가 요청을 처리하지 못한 것이 아니라,  
느린 요청이 누적되며 **큐 포화(Queue Saturation)** 상태에 도달했음을 의미한다.

---

## 6. 응답 시간 급락 구간에 대한 해석

응답 시간이 일시적으로 감소하는 구간은  
처리 실패가 아닌,  
누적된 요청이 한 번에 정리되며 발생한 현상으로 판단된다.

이후 응답 시간이 높은 수준에서 유지된 것은  
서버가 요청을 지속적으로 처리하고 있으나  
사용자 관점에서는 정상적인 서비스 경험을 제공하지 못하고 있음을 의미한다.

---

## 7. 결론

Mixed Traffic Test를 통해  
소수의 느린 요청이 전체 시스템 성능과  
SLA 만족 여부에 결정적인 영향을 미칠 수 있음을 확인하였다.

본 테스트는 평균 응답 시간만으로는  
성능 상태를 정확히 판단할 수 없음을 보여주는  
대표적인 사례로 활용 가능하다.
