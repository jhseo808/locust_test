# Spike Load Test 결과 분석

## 1. 테스트 개요

본 테스트는 갑작스러운 트래픽 증가 상황에서  
시스템의 처리 능력과 부하 제거 후 회복 여부를 검증하기 위해 수행되었습니다.

- 테스트 대상: https://httpbin.org
- 테스트 유형: Spike Test (LoadTestShape 사용)
- 목적:
  - 트래픽 급증 시 시스템 안정성 확인
  - 성능 저하 및 회복 여부 검증

---

## 2. 사용자 부하 시나리오

- 정상 부하 상태에서 갑작스럽게 사용자 수 증가
- Spike 구간 종료 후 원래 부하로 복귀
- 실제 서비스 장애 상황을 가정한 시나리오

---

### 3.1 요청 처리량 (RPS)

- Spike 구간에서 RPS는 **약 90~105**까지 급증
- 처리량은 Spike 동안에도 안정적으로 유지됨
- 부하 감소 후 즉시 정상 수준으로 복귀

→ 요청 적체나 처리 불능 상태는 발생하지 않음

---

### 3.2 응답 시간 변화

#### 50% 응답 시간 (Median)

- Spike 구간에서 약 **1.8~2.2초** 수준으로 상승
- 부하 감소 후 다시 정상 범위로 회복

#### 95% 응답 시간 (SLA 기준)

- Spike 구간에서 **3.2~3.5초**까지 상승
- 사전에 정의한 SLA(3초)를 초과하는 구간 확인
- 부하 해소 후 응답 시간 정상화

---

### 3.3 실패율 분석

- 전체 실패율: 약 **4%**
- 실패 발생 시점:
  - Spike 구간에 집중
- 실패 원인:
  - 서버 오류가 아닌
  - 응답 시간 3초 초과에 따른 **SLA 위반**

---

## 4. 회복(Recovery) 검증

- 사용자 수 감소 이후:
  - 응답 시간 정상화
  - 실패율 급감
- 지속적인 성능 저하나 장애 현상 없음

→ 시스템이 Spike 상황 이후 **정상 상태로 회복 가능함을 확인**

---

## 5. 결론

- 시스템은 갑작스러운 트래픽 증가에도 장애 없이 요청을 처리함
- Spike 상황에서는 일부 SLA 위반이 발생
- 부하 제거 후 성능은 정상 수준으로 회복됨

본 테스트를 통해 시스템의 **탄력성(Elasticity)** 과 **안정성**을 검증할 수 있었음
