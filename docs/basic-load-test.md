# Basic Load Test 결과 분석

## 1. 테스트 개요

본 테스트는 Locust를 이용하여  
**일정한 부하(steady state)** 상황에서 시스템의 기본 성능과 안정성을 검증하기 위해 수행되었습니다.

- 테스트 대상: https://httpbin.org
- 테스트 유형: Basic Load Test
- 동시 사용자 수: 100명
- 목적:
  - 일정 부하 환경에서의 처리량 확인
  - 응답 시간 및 실패율 기준선(Baseline) 확보

---

## 2. 사용자 부하 패턴

- 사용자는 점진적으로 증가한 뒤 100명으로 고정
- 테스트 종료 시까지 동일한 부하 유지
- 일반적인 운영 환경의 트래픽 패턴을 가정

---

### 3.1 요청 처리량 (RPS)

- 초당 요청 수(RPS)는 **약 45~48** 수준에서 안정적으로 유지됨
- 테스트 진행 시간 동안 급격한 변동이나 감소 현상 없음

→ 시스템이 일정 부하를 안정적으로 처리하고 있음을 확인

---

### 3.2 응답 시간

- 50% 응답 시간(Median): 약 **250~350ms**
- 95% 응답 시간: 약 **2.3~2.8초**

대부분의 사용자는 빠른 응답을 경험했으며,  
느린 응답은 의도적으로 지연된 API 호출로 인한 영향으로 판단됨

---

### 3.3 실패율

- 전체 실패율: 약 **1%**
- 실패 원인:
  - HTTP 오류가 아닌
  - 테스트 코드에서 정의한 **SLA(응답 시간 3초 초과)** 기준 위반

---

## 4. 결론

- 동시 사용자 100명 환경에서 시스템은 안정적으로 동작함
- 처리량 저하나 성능 열화 현상은 관찰되지 않음
- SLA 기준에 따른 실패가 정상적으로 감지됨

본 테스트는 이후 Spike Test를 위한 **성능 기준선(Baseline)** 으로 활용 가능함
